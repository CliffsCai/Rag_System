# 切片检索测试功能使用指南

## 功能位置

在文档管理页面(`DocumentManager.vue`)的顶部,有一个"🔍 切片检索测试"卡片。

## 功能说明

这是一个用于测试知识库检索效果的工具,可以直接查看检索返回的切片信息。

## 使用方法

### 1. 输入查询内容
在"查询内容"输入框中输入你想要检索的内容,例如:
- "什么是向量数据库"
- "如何配置ADB"
- "error: ECONNREFUSED"

### 2. 设置返回数量
默认返回50个结果,可以根据需要调整(1-100)。

### 3. 设置过滤条件(可选)
如果需要过滤特定文档,可以使用SQL WHERE格式,例如:
- `title = 'README'`
- `file_name LIKE '%.md'`
- `category = '技术' AND author = '张三'`

### 4. 点击"开始检索"
系统会调用后端API进行检索,并显示结果。

## 检索结果说明

结果表格包含以下列:

### ID
切片的唯一标识符

### 文件名
切片所属的文档文件名

### Rerank分数
重排序后的分数,分数越高表示相关性越强:
- 绿色(≥5): 高度相关
- 橙色(3-5): 中度相关
- 红色(<3): 低度相关

### 原始分数
检索时的原始相似度分数

### 检索来源
- 向量: 通过向量检索找到
- 全文: 通过全文检索找到
- 混合: 通过混合检索找到

### 内容预览
切片内容的前100个字符

### 展开查看
点击行首的展开按钮可以查看:
- 完整内容
- 元数据
- 文件URL(如果有)

## 检索参数配置

后端使用以下参数进行检索:
- **TopK**: 50
- **RerankFactor**: 1.0
- **HybridSearch**: Weight
- **HybridSearchArgs**: `{"Weight": {"alpha": 0.5}}` (混合检索,向量和全文各占50%)
- **IncludeFileUrl**: true

## 排序规则

结果按以下优先级排序:
1. 如果有Rerank分数,按Rerank分数从高到低
2. 否则按原始分数从高到低

## API端点

```
POST /api/v1/documents/search
```

**请求参数**:
- `query`: 查询文本(必填)
- `top_k`: 返回数量(默认50)
- `filter`: 过滤条件(可选)

**响应格式**:
```json
{
  "success": true,
  "message": "检索成功",
  "data": {
    "query": "查询内容",
    "results": [
      {
        "id": "chunk-id",
        "file_name": "document.pdf",
        "content": "切片内容...",
        "score": 0.85,
        "rerank_score": 6.23,
        "retrieval_source": 3,
        "metadata": {},
        "file_url": "https://..."
      }
    ],
    "total": 50
  }
}
```

## 使用场景

1. **测试检索效果**: 验证知识库是否正确索引了文档
2. **调试查询**: 查看不同查询返回的结果质量
3. **验证过滤**: 测试过滤条件是否正确工作
4. **分析相关性**: 通过Rerank分数判断结果相关性

## 注意事项

1. 检索前需要先上传文档到知识库
2. 过滤条件使用SQL WHERE语法,语法错误会导致检索失败
3. 返回数量越大,检索时间越长
4. Rerank分数是重排序后的分数,比原始分数更准确

## 故障排查

### 问题: 检索返回0个结果
- 检查知识库是否有文档
- 检查查询内容是否过于具体
- 尝试使用更通用的关键词

### 问题: 检索失败
- 检查后端服务是否运行
- 检查过滤条件语法是否正确
- 查看浏览器控制台的错误信息

### 问题: 结果相关性低
- 尝试调整查询内容
- 检查文档是否正确上传
- 查看Rerank分数判断相关性
